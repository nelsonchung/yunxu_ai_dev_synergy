# 使用說明文件

本文件整理平台主要功能與操作流程，提供客戶、開發者與管理員快速了解如何使用系統。

## 快速入門

### 客戶 5 分鐘上手
1. 註冊帳號：前往「登入/註冊」頁面完成註冊。
2. 提交需求：在「提交需求」填寫需求背景、目標、範圍、預算與時程。
3. 等待報價：開發者完成文件後會產生報價，通知會出現在「訊息中心」。
4. 簽核與追蹤：在「我的需求」完成簽核、審核報價與追蹤進度。

### 開發者 5 分鐘上手
1. 註冊並申請角色：註冊後聯繫管理員申請開發者角色。
2. 查看需求：在「需求中心」查看客戶需求清單。
3. 撰寫文件與報價：完成系統架構與軟體設計文件並提交報價。
4. 開發與驗證：在「專案工作台」更新開發清單與驗證進度。


## 角色與權限

### 客戶
- 提交需求、審核報價、簽核文件、追蹤進度。
- 主要入口：「我的需求」。

### 開發者
- 撰寫系統架構、軟體設計、測試與使用說明文件。
- 產生報價並更新開發 / 驗證清單。
- 主要入口：「專案工作台」。

### 管理員
- 管理使用者與權限，協調需求與客服。
- 可查看稽核記錄、處理客服訊息。

## 完整流程

1. **需求階段**
   - 客戶提交需求 → 管理員審核 → 指派開發者。

2. **文件階段**
   - 開發者撰寫系統架構與軟體設計 → 客戶簽核。
   - 系統驗證文件撰寫 → 客戶簽核。
   - 簽核後自動展開開發清單與驗證清單。

3. **報價階段**
   - 開發者基於軟體設計文件填寫報價。
   - 客戶審核報價（同意或提出修改）。

4. **開發階段**
   - 依開發清單逐項完成。
   - 在「專案工作台」更新開發進度。

5. **驗證階段**
   - 依驗證清單逐項測試。
   - 在「專案工作台」更新驗證進度。

6. **交付階段**
   - 開發者提交「使用說明文件」。
   - 客戶簽核後專案結案。

## 文件與簽核

### 文件類型
- 需求規格文件（客戶填寫，管理員審核）
- 系統架構文件（開發者撰寫，客戶簽核）
- 軟體設計文件（開發者撰寫，客戶簽核）
- 測試文件（開發者撰寫，客戶簽核）
- 使用說明文件（開發者撰寫，客戶簽核）

### 簽核狀態
- 草稿：開發者撰寫中
- 待簽核：送出待審
- 已核准：可進入下一階段
- 退回修改：需調整後重新送審

### 簽核入口
- 客戶：在「我的需求」的「文件與簽核」分頁
- 開發者：在文件編輯器完成後送出審核
- 所有簽核動作會記錄在稽核紀錄（管理員可查看）

## 狀態說明

### 需求狀態
- 草稿 / 待審核 / 進行中 / 已完成 / 已取消

### 報價狀態
- 待審核 / 已核准 / 需修改

### 通知類型
- 文件待簽核
- 報價待審核
- 簽核結果
- 留言通知
- 進度更新

## 常見問題

### 如何修改已提交的需求？
若需求已送審，請在留言區說明修改，管理員會協助調整。

### 報價退回後怎麼辦？
開發者會收到通知並依建議調整後重新提交。

### 交付後發現問題怎麼辦？
請透過「客服中心」回報，管理員會協助安排修正。
