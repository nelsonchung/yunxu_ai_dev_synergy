# 客戶需求文件（Customer Requirement）

## 1. 文件目的
本文件用於描述「鋆旭 AI-Dev 軟體系統媒合平台」的客戶需求，作為需求澄清、媒合評估、協作開發與品質交付的共識依據。

## 2. 產品定位與核心理念
- 產品定位：以「CMMI 標準流程 + AI 協作」為核心的軟體系統媒合平台，提供需求澄清、開發媒合與交付品質管理。
- 核心理念：以可重複、可追溯的流程串起需求、開發與交付，導入 AI 工具提升效率與品質。

## 3. 目標使用者與價值主張
- 客戶（發案方）：需求清楚、進度透明、交付可驗證，降低反覆溝通成本。
- 合作開發團隊：AI 輔助開發與文件撰寫，提升產能並聚焦核心功能。
- 管理者：可量化的里程碑與品質指標，便於風險控管與決策。

## 4. 需求目標
- 需求對齊：透過標準化規格文件與訪談，降低資訊落差。
- 媒合評估：以需求文件為基礎，完成技術、預算與時程三軸匹配。
- 協作開發：依系統開發文件與軟體設計文件，導入 AI 協作提升效率。
- 品質交付：AI Code Review + 測試檢核報告，確保交付品質。

## 5. 核心流程需求
### 5.1 平台流程（對外可見）
1. 需求對齊：客戶訪談、需求討論並產生需求文件。
2. 媒合評估：依需求文件建立系統開發文件，匹配技術/預算/時程。
3. 協作開發：依系統開發文件建立軟體設計文件，AI 協助實作項目條列、紀錄與追蹤。
4. 品質交付：依軟體設計文件建立系統測試文件，提供 AI Code Review 與測試檢核報告。

### 5.2 CMMI 對齊流程（內部管理）
1. 需求管理：收斂範圍，建立規格文件與確認流程。
2. 專案規劃：定義里程碑、風險評估、資源與時程。
3. 工程實作：任務拆分、版本控管、開發協作。
4. 驗證與測試：AI 輔助測試、品質檢核與缺陷追蹤。
5. 交付與驗收：交付文件齊全、可維護與可延展。
6. 持續改善：回饋分析與流程優化。

## 6. 功能需求（高層級）
- 需求文件化：支援需求訪談摘要與規格文件產出。
- 估工與媒合：依需求文件提供技術、預算、時程的匹配評估。
- 設計文件化：依系統開發文件產出軟體設計文件。
- 協作追蹤：實作項目條列化、紀錄與追蹤。
- 測試與檢核：產出系統測試文件，提供 AI Code Review 與測試檢核報告。
- 交付文件整理：交付文件自動整理與版本記錄。

## 7. AI 協作需求
- 需求訪談摘要與文件撰寫。
- 需求拆解與功能清單化。
- AI 程式碼開發與審查、規範檢核。
- 測試案例生成與品質報告。
- 交付文件自動整理與版本記錄。

## 8. 非功能需求（方向性）
- 可追溯：各階段文件與決策可被追蹤與驗證。
- 可視化：里程碑與驗收條件清楚可見。
- 品質控管：AI + 人工雙重驗證，提升交付可靠性。

## 9. 合作模式
- 專案媒合：一次性專案，清楚範疇與里程碑交付。
- 長期技術夥伴：團隊協作模式支援持續迭代。
- AI 加值支援：導入 AI 工作流，加速開發與驗證。

## 10. 成功指標（定性）
- 客戶需求文件清晰且雙方共識一致。
- 里程碑與驗收節奏可控、可視化。
- 交付版本經過 AI + 人工品質驗證。

## 11. 範圍限制與假設
- 本文件聚焦於需求與流程定義，不包含具體技術實作細節。
- 成功交付依賴客戶需求資訊完整與合作團隊配合流程。
